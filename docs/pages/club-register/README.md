# 俱乐部注册产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.5

更新时间：2026-04-22

## 页面目标

俱乐部注册页用于创建一个可被平台识别的俱乐部主体，并把新主体接入“发布身份”链路，供用户后续以俱乐部身份发布活动。

当前版本目标：

- 支持创建俱乐部基础资料
- 支持上传封面和 Logo
- 支持地图选点回填俱乐部地点
- 支持填写联系人与联系方式
- 支持从创建活动页进入并在完成后返回创建活动
- 支持注册成功后自动选中新注册俱乐部身份
- 支持从“我的”入口创建成功后进入俱乐部管理继续维护

## 在主链路中的位置

该页面位于主链路中的“发布身份补齐”步骤：

`登录 -> 进入创建活动页 -> 选择俱乐部卡 -> 俱乐部注册/完善页 -> 返回创建活动并刷新发布身份 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

它作为独立页面承接创建和完善动作；从创建活动链路进入时，保存成功后返回创建活动并刷新俱乐部发布身份。

注册成功后的持续维护由 [俱乐部管理](../club-management/README.md) 承接。

## 前置条件

- 当前用户已登录
- 当前用户尚无可用俱乐部发布身份，或当前俱乐部资料未满足可发布条件，或主动从“我的”入口创建俱乐部
- 登录主体、发布身份、俱乐部权限口径统一引用 [角色边界说明](../../global/role-boundaries.md)

## 页面职责

- 创建俱乐部主体资料
- 创建俱乐部地点与联系方式
- 在成功后返回来源页，并透传新建 `clubId`
- 在来源为 `activity-create` 时，支持上层页面自动切换为新俱乐部身份，并展开后续活动表单
- 在来源为 `profile` 时，支持进入俱乐部管理页继续维护资料

## 非职责边界

- 不负责创建活动表单细节
- 不负责活动发布后的详情与报名
- 不负责注册后的俱乐部资料持续管理
- 不负责俱乐部成员管理
- 不负责俱乐部审核后台
- 不负责俱乐部主页装修

## 入口与回跳

建议入口：

- `activity-create` 中没有可用俱乐部身份时点击 `注册俱乐部`
- `profile` / “我的”中主动创建俱乐部

成功后的回跳规则：

- 来源为 `activity-create`
  - 返回创建活动页
  - 带回 `clubId`
  - 自动选中新注册俱乐部作为发布身份
  - 先前已填写的活动表单内容保持不变
- 来源为 `profile`
  - 进入 [俱乐部管理](../club-management/README.md)
  - 默认选中新注册俱乐部
  - 用户可继续编辑资料或进入俱乐部维度的场馆与场地管理

## 核心模块

### 1. 页面标题区

- 标题：`注册俱乐部`
- 副标题：`完善俱乐部资料后，即可作为俱乐部主体使用`

### 2. 基础资料区

- 俱乐部封面
- 俱乐部 Logo
- 俱乐部名称
- 俱乐部品类
- 俱乐部简介

### 3. 位置信息区

- 地图选点入口
- 地点名称
- 区域
- 详细地址

### 4. 联系方式区

- 联系人
- 联系电话
- 微信号

### 5. 底部操作区

- `返回`
- `提交注册`

## 字段与状态

### 字段

- `coverUrl`
  - 单图上传
  - 选填
- `logoUrl`
  - 单图上传
  - 选填
- `name`
  - 单行输入
  - 必填
  - 建议长度 2 到 30 字
- `category`
  - 单选 picker
  - 当前版本固定为 `BADMINTON`
- `description`
  - 多行输入
  - 选填
- `province / city / district / address / latitude / longitude`
  - 地图选点回填地点名、详细地址和内部定位坐标
  - `province / city / district` 可由选点结果补充，或允许手动修正
  - 俱乐部地点必填
- `contactName`
  - 必填
- `contactPhone`
  - 必填，需做手机号格式校验
  - 本轮作为俱乐部可发布的硬性前置字段
- `wechatId`
  - 选填

### 状态

- 俱乐部状态枚举不在页面内重新定义
- 当前 v1 默认提交成功后直接进入可用状态，后续如接审核流，再按全局文档扩展

## 关键交互

- 页面既可从“我的”独立打开，也可从创建活动页进入后返回来源
- 页面初始展示空表单，品类默认选中羽毛球
- 图片上传成功后即时回显，可替换
- 地图选点默认使用微信小程序原生 `wx.chooseLocation`
- 地图选点完成后自动回填地点名、地址和内部经纬度
- `province / city / district` 若未完整返回，可由用户手动补充或修正
- 点击 `提交注册` 时：
  1. 校验必填项
  2. 校验手机号格式
  3. 创建俱乐部
  4. 自动创建当前用户为该俱乐部 `OWNER`
  5. 按来源页回跳

## 异常与空状态

- `请填写俱乐部名称`
- `俱乐部名称长度需为 2-30 字`
- `请选择俱乐部品类`
- `请选择俱乐部地点`
- `请填写联系人`
- `请填写联系电话`
- `联系电话格式不正确`
- 图片上传失败或地图返回失败时，保留当前表单内容，不清空已填项

## 验收标准

- 用户可成功创建一个俱乐部主体
- 用户从创建活动页进入时，注册成功后可返回创建活动页
- 返回创建活动页后，新俱乐部会被自动选中为发布身份
- 创建活动页中用户先前填写的表单内容不会丢失
- 用户在当前页只看到地点名和地址，不看到经纬度文案或数值
- 俱乐部注册页不再展开创建活动页的完整规则，只承接身份创建与返回
- 从“我的”入口创建成功后，可进入俱乐部管理继续维护该俱乐部
