# 首页产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-24

## 页面目标

首页用于承接游客和已登录用户的公共浏览入口，优先展示可参与活动，并提供搜索、分类筛选、创建活动和进入详情的起点。

当前版本目标：

- 支持游客浏览活动推荐
- 支持按项目分类和关键词搜索活动
- 支持进入活动详情
- 支持从首页进入创建活动

## 在主链路中的位置

首页位于 MVP 主链路的最前置浏览入口：

`首页 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

用户无需先登录或建档即可浏览首页内容；当用户点击创建活动、报名等角色相关动作时，再进入登录和资料补齐流程。

## 前置条件

- 小程序已启动并可读取活动列表数据
- 游客态可访问公开活动列表
- 登录、建档、报名和创建活动的拦截规则统一引用 [用户流程](../../global/user-flow.md)

## 页面职责

- 展示活动推荐列表
- 提供活动搜索和分类筛选入口
- 提供进入活动详情的入口
- 提供进入创建活动的入口
- 保持游客浏览和动作前补齐资料的边界清晰

## 非职责边界

- 不负责完整活动详情展示
- 不直接承接报名或取消报名
- 不负责“我的活动”集合回收
- 不负责俱乐部、场馆和场地主数据维护

## 入口与回跳

建议入口：

- 小程序默认首页
- 底部 tab 的 `首页`

跳转规则：

- 点击活动条目进入 [活动详情](../activity-detail/README.md)
- 点击创建活动进入 [创建活动](../activity-create/README.md)，由创建页继续承接登录与角色分流
- 从活动详情返回时，回到首页当前浏览上下文

## 核心模块

### 1. 顶部搜索区

- 搜索关键词
- 筛选入口
- 当前筛选状态摘要

### 2. 分类区

- `全部`
- `羽毛球`
- 后续新增项目时沿用同一分类模型

### 3. 活动推荐列表

每个活动卡片至少展示：

- 活动标题
- 发布主体
- 场馆摘要
- 时间摘要
- 费用摘要
- 已报名人数摘要
- 进入详情动作

### 4. 创建入口

- 固定或显著的创建活动入口
- 点击后进入创建活动页，不在首页内展开创建表单

## 字段与状态

### 展示字段

- `id`
- `title`
- `ownerLabel`
- `venueLabel`
- `scheduleText`
- `chargeText`
- `confirmedCount`
- `waitlistCount`
- `coverUrl`

### 状态说明

- 首页不定义新的活动状态
- 活动生命周期、报名状态和权限口径统一引用全局文档
- 搜索和分类只影响当前页展示，不改变活动本身状态

## 关键交互

### 浏览活动

1. 用户进入首页。
2. 页面加载公开活动列表。
3. 用户查看推荐活动卡片。
4. 用户点击活动卡片或 `查看详情`。
5. 页面跳转到活动详情页。
6. 用户从详情返回时，首页保留当前搜索和分类上下文。

### 搜索与分类

1. 用户输入关键词。
2. 页面按活动标题、场馆和发布主体做本地或服务端过滤。
3. 用户切换分类。
4. 页面叠加分类条件刷新活动卡片。
5. 若无匹配结果，展示空状态和清空筛选入口。

### 创建活动入口

1. 用户点击创建活动入口。
2. 页面跳转到创建活动页。
3. 创建活动页执行登录、角色卡和资料补齐判断。
4. 若用户取消登录或补齐流程，返回首页或停留来源页。

## 异常与空状态

- 活动列表为空：提示暂无活动，并保留创建活动入口
- 搜索无结果：提示未找到匹配活动
- 列表加载失败：提示重试
- 筛选功能未完整实现时：展示建设中提示，不影响活动列表浏览

## 验收标准

- 游客可进入首页并浏览活动列表
- 用户可按分类和关键词筛选活动
- 点击活动可进入活动详情页
- 点击创建活动可进入创建活动页，并由后续页面承接登录与资料补齐
- 首页不直接承担报名、取消报名和活动管理动作
