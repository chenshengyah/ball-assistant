# 俱乐部管理产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-24

## 页面目标

俱乐部管理页用于承接用户从“我的”进入后的俱乐部查看、资料编辑和俱乐部维度场馆管理入口，补齐“创建俱乐部后如何继续维护”的闭环。

当前版本目标：

- 展示当前用户拥有的俱乐部列表
- 展示俱乐部资料完整度
- 支持查看和编辑俱乐部基础资料
- 支持维护封面、Logo、地点、联系人和手机号
- 支持进入俱乐部维度的场馆与场地管理

## 在主链路中的位置

该页面位于“我的”入口下的俱乐部维护分叉：

`我的 -> 俱乐部入口 -> 俱乐部管理 -> 编辑资料 / 管理场馆与场地 -> 返回我的或创建活动`

它不阻塞最小“创建活动 -> 报名活动”闭环，但用于补齐俱乐部主体创建后的持续维护能力。

## 前置条件

- 当前用户已登录
- 当前用户已完成基础资料补齐
- 当前用户至少拥有一个俱乐部 `OWNER` 身份
- 俱乐部权限口径统一引用 [角色边界说明](../../global/role-boundaries.md)

## 页面职责

- 展示当前用户拥有的俱乐部
- 展示俱乐部资料完整度和可发布状态
- 编辑俱乐部基础资料、地点和联系方式
- 进入俱乐部 owner 维度的场馆与场地管理
- 保存成功后刷新创建活动页可用发布身份和俱乐部卡状态

## 非职责边界

- 不负责创建活动表单
- 不负责活动发布后的详情和报名
- 不负责俱乐部成员管理
- 不负责俱乐部审核后台
- 不负责俱乐部主页装修
- 不负责复杂经营统计

## 入口与回跳

建议入口：

- `profile` / “我的”中的俱乐部入口
- `club-register` 从“我的”来源创建成功后的回跳
- 后续创建活动页中“完善俱乐部资料”可复用本页字段与保存能力

跳转规则：

- 若用户没有俱乐部，入口进入 [俱乐部注册](../club-register/README.md)
- 若用户已有一个俱乐部，入口直接进入该俱乐部管理页
- 若用户已有多个俱乐部，先展示俱乐部列表，再进入选中俱乐部详情
- 点击 `管理场馆与场地` 进入 [场馆与场地管理](../venue-court-management/README.md)，并携带 `ownerType = CLUB` 与当前 `clubId`
- 从场馆与场地管理返回后，刷新俱乐部关联的场馆摘要
- 从本页返回时，回到“我的”页或来源页面

### 状态说明

- 当前版本只展示当前用户拥有的 `OWNER` 俱乐部
- `contactPhone` 是俱乐部可用于发布活动的硬性前置字段
- 地图经纬度继续存储，但页面只向用户展示地点名、地址和省市区

## 关键交互

### 进入俱乐部管理

1. 用户从“我的”点击俱乐部入口。
2. 页面检查登录和基础资料完整度。
3. 若资料未完善，先进入完善信息流程。
4. 页面调用 `GET /api/clubs/my`。
5. 若无俱乐部，进入俱乐部注册页。
6. 若有一个俱乐部，进入该俱乐部管理详情。
7. 若有多个俱乐部，展示俱乐部列表并等待用户选择。

### 编辑俱乐部资料

1. 用户进入俱乐部管理详情。
2. 页面回填俱乐部基础资料、地点和联系方式。
3. 用户修改封面、Logo、名称、简介、地点或联系人信息。
4. 用户点击保存资料。
5. 页面校验必填项和手机号格式。
6. 页面调用 `PUT /api/clubs/:clubId`。
7. 保存成功后刷新俱乐部资料和可发布状态。

### 管理俱乐部场馆与场地

1. 用户在俱乐部管理页点击 `管理场馆与场地`。
2. 页面进入场馆与场地管理页。
3. 目标页面使用 `ownerType = CLUB` 和当前 `clubId` 查询场馆。
4. 用户新增或编辑场馆与场地。
5. 保存成功后返回俱乐部管理页。
6. 俱乐部管理页刷新场馆摘要。

### 注册成功回跳

1. 用户从“我的”进入俱乐部注册页。
2. 用户提交俱乐部注册信息。
3. 注册成功后返回俱乐部管理页。
4. 俱乐部管理页默认选中新注册俱乐部。
5. 用户可继续编辑资料或进入场馆与场地管理。

## 异常与空状态

- 无俱乐部：提示创建俱乐部，并提供注册入口
- 俱乐部列表加载失败：提示重试
- 保存失败：保留当前表单内容并提示失败原因
- 图片上传失败：保留已填资料，不清空表单
- 地图返回失败：保留当前地点资料，不清空表单
- 当前用户非 `OWNER`：不展示编辑入口

## 验收标准

- 已有俱乐部用户可从“我的”进入俱乐部管理页
- 无俱乐部用户会进入俱乐部注册流程
- 用户可查看并编辑俱乐部基础资料、地点和联系方式
- 保存俱乐部资料后，可发布状态会刷新
- 用户可从俱乐部管理页进入俱乐部维度的场馆与场地管理
- 本页不承接成员管理、审核后台或俱乐部主页装修
