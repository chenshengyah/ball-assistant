# 我的页面产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-24

## 页面目标

我的页是个人入口容器，用于展示游客态、登录态和资料完整度，并承接完善资料、我的活动、创建活动和俱乐部入口。

当前版本目标：

- 支持游客态说明
- 支持展示资料完整度
- 支持进入完善信息
- 支持进入我的活动
- 支持进入创建活动
- 支持进入俱乐部入口，并按俱乐部状态分流到注册或管理

## 在主链路中的位置

我的页是 MVP 主链路的入口回收容器：

`我的 -> 完善信息 / 我的活动 / 创建活动 / 俱乐部注册 / 俱乐部管理`

它不替代具体业务页面，只负责展示当前账号状态并把用户送到对应流程。

## 前置条件

- 小程序可读取当前登录状态
- 系统可判断基础资料是否完整
- 我的活动和俱乐部入口需复用动作前资料补齐规则
- 系统可读取当前用户拥有的俱乐部列表

## 页面职责

- 展示游客、已登录未完善、已完善资料三种状态
- 提供资料补齐或编辑入口
- 提供我的活动入口
- 提供创建活动入口
- 提供俱乐部入口，并按状态进入注册或管理

## 非职责边界

- 不负责完整资料表单提交
- 不负责我的活动列表展示
- 不负责创建活动表单
- 不负责俱乐部注册表单
- 不负责俱乐部资料编辑表单
- 不负责活动详情和活动管理动作

## 入口与回跳

建议入口：

- 底部 tab 的 `我的`
- 角色相关动作取消后返回个人入口

跳转规则：

- 点击完善资料进入 [完善信息 / 资料补全](../user-registration/README.md)
- 点击我的活动进入 [我的活动](../my-activities/README.md)
- 点击创建活动进入 [创建活动](../activity-create/README.md)
- 点击俱乐部入口时，若无俱乐部进入 [俱乐部注册](../club-register/README.md)
- 点击俱乐部入口时，若已有俱乐部进入 [俱乐部管理](../club-management/README.md)
- 子页面返回时，我的页刷新登录状态和资料完整度

## 核心模块

### 1. 账号状态区

- 昵称或游客名称
- 头像占位或用户头像
- 游客态 / 待完善 / 已完善状态标签
- 资料完整度说明

### 2. 资料入口区

- `登录并完善资料`
- `编辑资料`
- 基础资料状态说明

### 3. 业务入口区

- `我的活动`
- `创建活动`
- `俱乐部`

### 4. 提示区

- 游客可浏览公开内容说明
- 角色相关动作需要登录和补资料说明

## 字段与状态

### 展示字段

- `nickname`
- `avatarUrl`
- `avatarColor`
- `baseProfileComplete`
- `contactProfileComplete`
- `isProfileComplete`
- `statusLabel`

### 状态说明

- 游客态只允许浏览公开页面
- 已登录但资料未完善时，角色相关入口先进入完善信息流程
- 已完善资料后，可继续我的活动、创建活动和俱乐部入口
- 无俱乐部用户进入俱乐部注册
- 已有俱乐部用户进入俱乐部管理

## 关键交互

### 完善资料

1. 用户进入我的页。
2. 页面读取当前登录状态和资料完整度。
3. 用户点击 `登录并完善资料` 或 `编辑资料`。
4. 页面进入完善信息页，并携带 `profile` 来源。
5. 保存成功后返回我的页。
6. 我的页刷新资料状态。

### 打开我的活动

1. 用户点击 `我的活动`。
2. 页面检查登录和基础资料完整度。
3. 若未满足条件，进入完善信息页。
4. 保存成功后继续打开我的活动。
5. 若已满足条件，直接进入我的活动页。

### 创建活动

1. 用户点击 `创建活动`。
2. 页面检查登录状态。
3. 登录通过后进入创建活动页。
4. 创建活动页继续处理角色、资料和发布前置条件。

### 俱乐部入口

1. 用户点击俱乐部入口。
2. 页面检查登录和基础资料完整度。
3. 若未满足条件，进入完善信息页。
4. 满足条件后，页面读取当前用户拥有的俱乐部列表。
5. 若用户没有俱乐部，进入俱乐部注册页。
6. 若用户已有俱乐部，进入俱乐部管理页。
7. 子页面返回后，我的页刷新资料状态和俱乐部入口状态。

## 异常与空状态

- 游客态：展示可继续浏览公开内容的说明
- 登录失败：保留当前页并提示稍后重试
- 资料保存失败：由完善信息页提示失败原因
- 我的活动入口被拦截：完成资料补齐后继续原动作
- 俱乐部列表加载失败：提示稍后重试
- 俱乐部入口尚未完整实现时：展示建设中提示，但保留资料补齐规则

## 验收标准

- 游客可进入我的页并看到游客态说明
- 已登录未完善用户可从我的页进入完善信息页
- 已完善用户可进入我的活动和创建活动
- 我的活动和俱乐部入口都复用动作前补资料规则
- 无俱乐部用户点击俱乐部入口会进入俱乐部注册
- 已有俱乐部用户点击俱乐部入口会进入俱乐部管理
- 我的页只作为入口容器，不展开具体业务表单
