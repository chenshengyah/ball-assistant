# 我的活动产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-13

## 页面目标

我的活动页用于回收当前用户相关的活动集合，统一展示“我发布的”和“我报名的”活动，并作为进入活动详情的列表入口。

当前版本目标：

- 区分“我发布的”和“我报名的”两类集合
- 统一展示活动生命周期摘要
- 支持按集合切换和基础筛选
- 点击活动条目进入活动详情

## 在主链路中的位置

该页面位于主链路末端的回收步骤：

`完善信息 -> 确认发布身份 -> 俱乐部注册 -> 创建活动（必要时补充场馆与场地） -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

它负责活动回看与集合管理，不替代活动详情页。

## 前置条件

- 当前用户已完成基础建档
- 当前用户至少创建过活动或报名过活动
- 我的活动集合定义统一引用全局文档

## 页面职责

- 展示“我发布的”活动列表
- 展示“我报名的”活动列表
- 按统一生命周期口径展示条目状态
- 为每个条目提供进入活动详情的入口

## 非职责边界

- 不负责完整活动详情展示
- 不直接承接报名、取消报名、调场、调容量、取消活动
- 不负责创建活动表单
- 不负责俱乐部与场馆主数据维护

## 入口与回跳

建议入口：

- `profile` / “我的”页中的 `我的活动`
- 创建活动成功后，用户可后续从“我的活动-我发布的”回看
- 报名成功后，用户可后续从“我的活动-我报名的”回看

跳转规则：

- 点击任一活动条目，统一进入 [活动详情](../activity-detail/README.md)
- 从活动详情返回时，应回到原集合和原滚动位置

## 核心模块

### 1. 页面标题区

- 标题：`我的活动`
- 副标题：`回看我发布的和我报名的活动`

### 2. 集合切换区

- `我发布的`
- `我报名的`

### 3. 活动列表区

每个条目至少展示：

- 活动标题
- 发布主体
- 场馆摘要
- 时间摘要
- 生命周期标签
- 当前用户报名状态摘要

### 4. 空状态区

- 当前集合为空时的引导文案

## 字段与状态

### 集合定义

- `published`
  - 我发布的活动
- `joined`
  - 我报名的活动

### 条目展示字段

- `id`
- `title`
- `ownerLabel`
- `venueLabel`
- `scheduleText`
- `lifecycleStatusLabel`
- `signupStatusLabel`
- `currentUserSignupLabel`

### 状态说明

- 条目生命周期统一引用 [活动全链路说明](../../global/activity-lifecycle.md)
- 页面不单独定义新的状态枚举

## 关键交互

- 默认进入最近一次访问的集合，或默认进入 `我发布的`
- 支持集合切换
- 可按基础生命周期筛选，如 `报名中 / 进行中 / 已结束 / 已取消`
- 点击条目进入活动详情页查看完整信息和动作
- 再次发布不在本页直接展开复杂表单，统一进入创建活动页

## 异常与空状态

- `我发布的` 为空：提示去创建活动
- `我报名的` 为空：提示去活动列表浏览活动
- 列表加载失败：提示重试

## 验收标准

- 页面至少区分“我发布的”和“我报名的”两类集合
- 两类集合中的条目都按统一生命周期展示
- 点击任一条目都能进入活动详情页
- 页面只做集合回收与跳转，不与活动详情职责重叠
