# 场馆与场地管理产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.2

更新时间：2026-04-13

## 页面目标

场馆与场地管理页用于维护活动发布依赖的场馆主数据和场地主数据，为创建活动提供可引用的场馆与场地来源。

当前版本目标：

- 支持新增和编辑场馆
- 支持地图选点回填场馆定位
- 支持新增和编辑场地
- 支持停用场地
- 支持创建活动时自动带出启用中的场地与默认人数

## 在主链路中的位置

该页面位于主链路中的“场馆/场地准备”步骤：

`用户建档 -> 选择发布身份 -> 俱乐部注册 -> 场馆/场地准备 -> 创建活动 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

它服务于创建活动前的主数据准备，不属于活动详情管理动作。

## 前置条件

- 当前用户已完成基础建档
- 当前用户具备个人活动创建权限，或具备俱乐部 `OWNER / ADMIN` 身份
- 场馆、场地、活动快照的资源边界统一引用全局文档

## 页面职责

- 维护场馆主数据
- 维护场地主数据
- 在创建活动时提供可选场馆和可选场地来源
- 保证停用主数据不影响历史活动快照

## 非职责边界

- 不负责活动详情中的调场
- 不负责活动详情中的调容量
- 不负责活动详情中的取消活动
- 不负责活动发布后的报名名单展示
- 不负责历史活动快照的反向修改

## 入口与回跳

建议入口：

- `activity-create` 页面中的 `管理场馆与场地`
- 后续 `profile` / “我的”中的场馆管理入口

回跳规则：

- 若从 `activity-create` 进入，保存成功后返回创建活动页
- 返回后刷新场馆列表与可选场地
- 若本次新增了新场馆，创建活动页默认切换到新建场馆
- 若编辑的是当前场馆，尽量保留当前表单中已启用的场地配置

## 核心模块

### 1. 页面标题区

- 标题：`场馆与场地管理`
- 副标题：`维护活动发布所需的场馆、定位和场地配置`

### 2. 场馆基础信息区

- 场馆名称
- 简称
- 导航名
- 地图定位入口
- 区域
- 详细地址

### 3. 场地列表区

- 场地卡片列表
- 场地号
- 场地名称
- 默认人数
- 默认费用
- 状态
- `新增场地`

### 4. 底部操作区

- `取消`
- `保存场馆`

## 字段与状态

### 场馆字段

- `name`
  - 必填
- `shortName`
  - 选填，不填写时默认同场馆名称
- `navigationName`
  - 选填，不填写时默认同场馆名称
- `province / city / district / address / latitude / longitude`
  - 由地图选点回填，可允许展示字段手动修正

### 场地字段

- `courtCode`
  - 必填
  - 同一场馆下唯一
- `courtName`
  - 必填
- `defaultCapacity`
  - 必填
  - 必须大于 0
- `defaultFeeCents`
  - 必填
  - 前端按元输入，提交时转分
- `status`
  - `ACTIVE / INACTIVE`

### 状态说明

- 页面不重新定义活动状态
- 场地停用只影响新活动可选项，不影响历史活动快照

## 关键交互

- 点击地图定位可拉起地图选点，并回填结构化地址
- 新建未保存场地可直接删除
- 已保存场地优先采用“停用”而不是硬删除
- 点击 `保存场馆` 时需要同时提交：
  - 场馆基础信息
  - 定位信息
  - 场地列表
- 保存成功后返回来源页，并刷新创建活动页中的可选数据

## 异常与空状态

- `请填写场馆名称`
- `请填写场馆地址`
- `请先选择场馆定位`
- `请至少新增一片场地`
- `请填写场地名称`
- `人数需大于 0`
- `费用不能小于 0`
- `场地序号重复，请刷新后重试`

## 验收标准

- 用户可成功新增一个带定位信息的场馆
- 用户可在一个场馆下新增多片场地
- 用户可停用已保存场地，且停用后不再出现在新建活动可选列表中
- 历史活动仍能显示发布时的场地快照
- 创建活动时可自动带出启用中的场地和默认人数
- 本页职责保持在“主数据准备”，不与活动详情中的管理动作混淆
