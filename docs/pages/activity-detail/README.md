# 活动详情产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.1

更新时间：2026-04-20

## 页面目标

活动详情页是活动发布后的主承接页，用于统一展示活动基础信息、图文详情、报名信息、当前用户状态和主办方管理动作。

当前版本目标：

- 展示活动基础信息
- 展示活动图文详情
- 展示报名模式与场馆 / 场地信息
- 展示报名名单与当前用户报名状态
- 承接报名与取消报名
- 承接调场、调容量、编辑图文详情、取消活动、再次发布

## 在主链路中的位置

该页面位于主链路中的发布后承接阶段：

`完善信息 -> 确认发布身份 -> 俱乐部注册 -> 创建活动（必要时补充场馆与场地） -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

无论用户是主办方还是报名者，发布后的主要操作都在本页完成。

## 前置条件

- 活动已成功创建或用户已从列表进入某个活动
- 活动状态、生命周期、动作守卫统一引用全局文档
- 报名、取消报名始终以个人身份处理

## 页面职责

- 展示活动详情、图文详情、发布主体和时间地点信息
- 展示两种报名模式下的报名摘要与报名名单
- 展示当前用户是否已报名、是否可取消报名
- 展示当前用户是否可管理该活动
- 统一承接报名、取消报名、调场、调容量、编辑图文详情、取消活动、再次发布

## 非职责边界

- 不负责活动创建输入
- 不负责“我的活动”集合回收
- 不负责场馆与场地主数据维护
- 不负责活动结束后的结算、评价、聊天

## 入口与回跳

建议入口：

- 创建活动发布成功后跳转进入
- 活动列表点击某个活动进入
- 我的活动列表点击某个活动进入

后续跳转：

- 再次发布：进入 [创建活动](../activity-create/README.md)
- 若来源为“我的活动”或活动列表，返回时回到原列表位置

## 核心模块

### 1. 活动摘要区

- 标题
- 发布主体
- 生命周期标签
- 报名状态标签
- 收费摘要

### 2. 时间与地点区

- 活动时间
- 取消截止时间
- 场馆名称
- `自主选场` 时的活动场地列表

### 3. 当前用户状态区

- 当前用户是否已报名
- 当前用户报名所在场地或统一报名状态
- 是否可取消报名

### 4. 图文详情区

- 展示活动图文详情富文本内容
- 支持文字段落与图片按顺序浏览
- 普通用户与已报名用户均只读查看

### 5. 报名区

- `统一分配 = GENERAL`
  - 统一报名按钮
  - 确认 / 候补统计
  - 统一报名名单
- 不提供系统内后续分配到具体场地的管理动作
- `自主选场 = USER_SELECT_COURT`
  - 各场地卡片
  - 每片场地的确认 / 候补人数
  - 每片场地报名按钮
  - 每片场地报名名单

### 6. 主办方管理区

- 编辑图文详情
- 调场
- 调容量
- 取消活动
- 再次发布

- 持久状态只引用 `DRAFT / PUBLISHED / CANCELLED`

- 页面不在本地重新发明状态枚举与动作守卫

## 关键交互

### 普通用户

- 未报名且仍可报名时：
  - `统一分配 / GENERAL` 可直接报名，不选择场地
  - `自主选场 / USER_SELECT_COURT` 选择具体场地报名
- 已报名时展示当前状态
- 在取消截止前可取消自己的报名

普通用户报名步骤：

1. 用户进入活动详情页。
2. 页面展示 `isSignupOpen`、当前用户报名状态和报名模式。
3. 用户点击报名。
4. 若用户未登录或资料未完善，先进入完善信息流程。
5. `GENERAL` 模式下，用户确认后直接提交报名。
6. `USER_SELECT_COURT` 模式下，用户先选择具体场地，再提交报名。
7. 若容量未满，系统返回确认报名状态。
8. 若容量已满且允许候补，系统返回候补状态和候补序号。
9. 页面刷新报名摘要、名单和当前用户状态。

取消报名步骤：

1. 已报名用户进入活动详情页。
2. 页面展示当前报名记录和取消截止时间。
3. 用户点击取消报名。
4. 页面校验 `canCancelCurrentUserRegistration`。
5. 若不可取消，按钮置灰并展示原因。
6. 若可取消，用户确认取消。
7. 系统取消报名记录。
8. 页面刷新报名摘要、名单和当前用户状态。

### 主办方

- 可查看并编辑活动图文详情
- 可查看完整报名情况
- `自主选场 / USER_SELECT_COURT` 下可调场
- 两种模式下都可按权限调容量
- 活动未结束前可取消活动
- 可从本页发起再次发布

编辑图文详情步骤：

1. 主办方进入活动详情页。
2. 页面校验 `permissions.canEditDetailContent`。
3. 主办方打开图文详情编辑入口。
4. 主办方调整文字和图片内容。
5. 主办方保存。
6. 页面留在活动详情页并刷新图文详情展示。

调场步骤：

1. 主办方进入 `USER_SELECT_COURT` 活动详情页。
2. 页面校验 `permissions.canMoveRegistration`。
3. 主办方在报名名单中选择一条有效报名记录。
4. 主办方选择目标场地。
5. 系统校验目标场地属于当前活动。
6. 系统移动报名记录，并按目标场地容量重新计算确认或候补状态。
7. 页面刷新场地报名名单和当前报名状态。

调容量步骤：

1. 主办方进入活动详情页。
2. 页面校验 `permissions.canAdjustCapacity`。
3. 主办方选择需要调整的活动容量或场地容量。
4. 主办方提交新容量。
5. 系统校验容量不能小于已确认人数。
6. 系统保存容量。
7. 页面刷新确认人数、候补人数和容量展示。

取消活动步骤：

1. 主办方进入活动详情页。
2. 页面校验 `permissions.canCancelActivity`。
3. 主办方点击取消活动。
4. 页面展示确认提示。
5. 主办方确认取消。
6. 系统将活动状态更新为 `CANCELLED`。
7. 页面留在活动详情页并展示已取消状态。

再次发布步骤：

1. 主办方进入活动详情页。
2. 页面校验 `permissions.canRepublish`。
3. 主办方点击再次发布。
4. 页面跳转创建活动页，并携带原活动 ID。
5. 创建活动页生成再次发布草稿。
6. 主办方修改必要信息后重新发布为新活动。
7. 原活动状态不变。

报名模式解释：

- `统一分配`：用户先报名，系统仅展示统一报名人数、候补和名单；后续分场由组织者线下处理
- `自主选场`：组织者先开放具体活动场地，用户报名时自行选择对应场地

图文详情编辑规则：

- `报名中 / 报名截止 / 进行中`：主办方可编辑
- `已结束 / 已取消`：主办方不可编辑
- 若未来开放 `DRAFT` 入口，则 `草稿` 沿用创建态继续编辑
- 编辑入口位于活动详情页的主办方管理区，不回跳创建活动页
- 编辑后的内容即时覆盖详情展示，不影响报名、调场、调容量等其他活动数据

## 异常与空状态

- 活动不存在：展示加载失败态与返回入口
- 活动已取消：不可再报名，展示取消结果
- 活动已结束：不可再报名或取消报名，展示结束态
- 报名截止：按钮置灰并展示原因
- 图文详情编辑失败：保留当前页上下文和已加载内容，并提示失败原因
- 场地或容量调整失败：保留当前页上下文并提示失败原因

## 验收标准

- 创建活动成功后统一跳转到本页
- 活动列表与我的活动都能统一进入本页
- `统一分配 / GENERAL` 与 `自主选场 / USER_SELECT_COURT` 两种模式都能在本页独立解释报名逻辑
- 活动图文详情可在本页对普通用户只读展示
- 普通用户能在本页完成报名和取消报名
- 主办方能在本页完成编辑图文详情、调场、调容量、取消活动、再次发布
- 本页不再被活动列表卡片或创建活动页替代
