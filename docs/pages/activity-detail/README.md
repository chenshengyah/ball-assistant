# 活动详情产品文档

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-13

## 页面目标

活动详情页是活动发布后的主承接页，用于统一展示活动基础信息、报名信息、当前用户状态和主办方管理动作。

当前版本目标：

- 展示活动基础信息
- 展示报名模式与场馆 / 场地信息
- 展示报名名单与当前用户报名状态
- 承接报名与取消报名
- 承接调场、调容量、取消活动、再次发布

## 在主链路中的位置

该页面位于主链路中的发布后承接阶段：

`用户建档 -> 选择发布身份 -> 俱乐部注册 -> 场馆/场地准备 -> 创建活动 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

无论用户是主办方还是报名者，发布后的主要操作都在本页完成。

## 前置条件

- 活动已成功创建或用户已从列表进入某个活动
- 活动状态、生命周期、动作守卫统一引用全局文档
- 报名、取消报名始终以个人身份处理

## 页面职责

- 展示活动详情、发布主体和时间地点信息
- 展示两种报名模式下的报名摘要与报名名单
- 展示当前用户是否已报名、是否可取消报名
- 展示当前用户是否可管理该活动
- 统一承接报名、取消报名、调场、调容量、取消活动、再次发布

## 非职责边界

- 不负责活动创建输入
- 不负责“我的活动”集合回收
- 不负责场馆与场地主数据维护
- 不负责活动结束后的结算、评价、聊天

## 入口与回跳

建议入口：

- 创建活动发布成功后跳转进入
- 活动列表点击某个活动进入
- 我的活动列表点击某个活动进入

后续跳转：

- 再次发布：进入 [创建活动](../activity-create/README.md)
- 若来源为“我的活动”或活动列表，返回时回到原列表位置

## 核心模块

### 1. 活动摘要区

- 标题
- 发布主体
- 生命周期标签
- 报名状态标签
- 收费摘要

### 2. 时间与地点区

- 活动时间
- 取消截止时间
- 场馆名称
- 按场地报名时的活动场地列表

### 3. 当前用户状态区

- 当前用户是否已报名
- 当前用户报名所在场地或统一报名状态
- 是否可取消报名

### 4. 报名区

- `GENERAL`
  - 统一报名按钮
  - 确认 / 候补统计
  - 统一报名名单
- `USER_SELECT_COURT`
  - 各场地卡片
  - 每片场地的确认 / 候补人数
  - 每片场地报名按钮
  - 每片场地报名名单

### 5. 主办方管理区

- 调场
- 调容量
- 取消活动
- 再次发布

### 状态说明

- 持久状态只引用 `DRAFT / PUBLISHED / CANCELLED`
- 生命周期与报名状态只引用全局派生规则
- 页面不在本地重新发明状态枚举与动作守卫

## 关键交互

### 普通用户

- 未报名且仍可报名时：
  - `GENERAL` 可直接报名
  - `USER_SELECT_COURT` 选择具体场地报名
- 已报名时展示当前状态
- 在取消截止前可取消自己的报名

### 主办方

- 可查看完整报名情况
- `USER_SELECT_COURT` 下可调场
- 两种模式下都可按权限调容量
- 活动未结束前可取消活动
- 可从本页发起再次发布

## 异常与空状态

- 活动不存在：展示加载失败态与返回入口
- 活动已取消：不可再报名，展示取消结果
- 活动已结束：不可再报名或取消报名，展示结束态
- 报名截止：按钮置灰并展示原因
- 场地或容量调整失败：保留当前页上下文并提示失败原因

## 验收标准

- 创建活动成功后统一跳转到本页
- 活动列表与我的活动都能统一进入本页
- `GENERAL` 与 `USER_SELECT_COURT` 两种模式都能在本页独立解释报名逻辑
- 普通用户能在本页完成报名和取消报名
- 主办方能在本页完成调场、调容量、取消活动、再次发布
- 本页不再被活动列表卡片或创建活动页替代
