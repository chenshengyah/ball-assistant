# 活动列表产品文档（历史结构参考）

面向对象：UI 设计、前端开发、后端开发

版本：v1.0

更新时间：2026-04-24

## 页面目标

活动列表能力已并入首页承接，本文件保留为历史结构参考。实际小程序不再注册独立 `activity` tab，也不再保留 `pages/activity/index` 路由。

当前版本目标：

- 支持游客态在首页浏览活动列表
- 支持展示当前用户参与、发布和可管理活动摘要
- 原活动页的场馆摘要能力不再作为公共 tab 展示
- 支持进入活动详情
- 支持进入创建活动和再次发布

## 在主链路中的位置

活动浏览位于公共浏览与角色动作之间：

`首页活动浏览 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

该页面可展示活动与场馆摘要，但所有报名、取消报名和主办方管理动作都进入活动详情页完成。

## 前置条件

- 活动列表数据可读取
- 游客态可浏览公开活动
- 创建活动和再次发布需通过登录与资料补齐规则

## 页面职责

- 展示活动列表和基础活动状态
- 展示当前用户活动统计摘要
- 展示场馆摘要信息
- 提供创建活动入口
- 提供再次发布入口
- 提供进入活动详情入口

## 非职责边界

- 不直接承接报名、取消报名、调场、调容量或取消活动
- 不负责完整活动详情展示
- 不负责场馆与场地主数据编辑
- 不负责“我的活动”集合回收

## 入口与回跳

当前实现口径：

- 底部 tab 仅保留 `首页 / 我的`
- 活动浏览入口在首页
- “我的”页中的活动相关入口进入我的活动或创建活动

跳转规则：

- 点击活动条目进入 [活动详情](../activity-detail/README.md)
- 点击创建活动进入 [创建活动](../activity-create/README.md)
- 点击再次发布进入创建活动页并携带来源活动 ID
- 从活动详情返回时，恢复来源列表位置和筛选上下文

## 核心模块

### 1. 用户摘要区

- 当前用户昵称或游客态文案
- 已发布活动数
- 可管理活动数
- 已报名活动数

### 2. 分段切换区

- `活动`
- `场馆`

### 3. 活动列表区

每个活动条目至少展示：

- 活动标题
- 发布主体
- 生命周期标签
- 报名状态标签
- 报名模式
- 场馆
- 时间
- 费用
- 当前用户报名状态

### 4. 场馆摘要区

- 场馆名称
- 地址摘要
- 场地数量
- 可作为创建活动时的资料来源说明

### 5. 操作入口区

- `创建活动`
- `查看详情`
- `再次发布`

## 字段与状态

### 活动展示字段

- `id`
- `title`
- `ownerLabel`
- `lifecycleStatusLabel`
- `signupStatusLabel`
- `signupMode`
- `venueLabel`
- `scheduleText`
- `chargeText`
- `cancelDeadlineText`
- `currentUserSignupLabel`
- `isManageable`

### 状态说明

- 页面只展示活动状态，不定义新的状态枚举
- 再次发布入口是否展示由 `isManageable` 和权限字段控制
- 报名和取消报名守卫统一引用全局状态矩阵

## 关键交互

### 浏览活动列表

1. 用户进入首页活动浏览区。
2. 页面加载活动列表。
3. 游客用户看到公共浏览文案和全部公开活动。
4. 已登录用户看到发布、管理和报名统计。
5. 用户点击活动条目。
6. 页面进入活动详情页。

### 切换活动和场馆

1. 用户点击分段控件。
2. 页面切换到对应内容区。
3. 切换不重置当前页面已加载数据。
4. 从详情返回后恢复切换前的分段状态。

### 创建活动

1. 用户点击 `创建活动`。
2. 页面触发登录检查。
3. 登录通过后进入创建活动页。
4. 创建活动页继续处理角色卡、资料补齐和正式表单。

### 再次发布

1. 用户在可管理活动条目上点击 `再次发布`。
2. 页面触发登录检查。
3. 页面跳转创建活动页，并携带 `sourceActivityId`。
4. 创建活动页按再次发布规则带出原活动配置。

## 异常与空状态

- 活动列表为空：提示暂无活动，并提供创建活动入口
- 场馆列表为空：提示暂无场馆资料
- 游客态无法执行创建或再次发布：进入登录与资料补齐流程
- 列表加载失败：提示重试

## 验收标准

- 游客可进入首页并浏览公开活动
- 已登录用户可看到活动统计摘要
- 历史活动/场馆分段仅作为结构参考，当前首页不再提供独立场馆公共浏览分段
- 点击活动条目可进入活动详情页
- 创建活动和再次发布不在本页展开复杂表单
- 从活动详情返回时可恢复原列表上下文
