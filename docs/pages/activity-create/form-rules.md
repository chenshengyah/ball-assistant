# 创建活动页字段规则

## 1. 发布身份

- 必选
- 页面顶部固定展示 `个人 / 俱乐部` 两张角色卡
- 使用身份卡片单选，不使用下拉框
- 角色未选定前，不展示后续活动表单
- `个人`：
  - 若基础资料未完善，打开资料补齐半屏
  - 若手机号未验证，打开资料补齐半屏并要求完成手机号验证
- `俱乐部`：
  - 若当前无可用俱乐部主体，打开俱乐部创建半屏
  - 若俱乐部资料不完整或缺少联系人手机号，打开俱乐部资料半屏

## 2. 活动封面

- 选填
- 支持单图上传
- 用于活动列表卡片和活动详情头图展示

## 3. 活动标题

- 必填
- 长度 1~30

## 4. 活动日期

- 必填
- 不能早于今天

## 5. 活动时间

- 开始时间必填
- 结束时间必填
- 结束时间必须晚于开始时间

## 6. 联系方式

- 创建页不再额外采集微信号
- `PERSONAL` 发布时，自动使用当前用户已验证手机号
- `CLUB` 发布时，自动使用俱乐部联系人手机号
- 个人未验证手机号或俱乐部缺少联系人手机号时，不可进入正式发布表单

## 7. 收费方式

- 必选
- 枚举：FREE / FIXED / AA / OTHER

## 8. 收费金额

- 仅当 chargeMode != FREE 时按创建页规则校验
- 条件必填
- 大于等于 0

## 9. 报名模式

- 必选
- 视觉上归属“报名与场馆区”
- 枚举：`GENERAL / USER_SELECT_COURT`
- `GENERAL` 对外显示为 `统一分配`
- `USER_SELECT_COURT` 对外显示为 `自主选场`
- 切换模式时，不清空公共字段

## 10. 场馆

- 必选
- 来自当前身份下可用场馆列表
- 支持在当前创建流程内新增场馆或编辑场馆定位
- 地图定位默认使用微信小程序原生 `wx.chooseLocation`
- 地图信息对用户只展示地点名和详细地址，不展示经纬度

## 11. 场地列表

- 仅当 signupMode = USER_SELECT_COURT 时显示
- 至少保留一片场地
- 支持新增、编辑、删除、调整顺序
- 提交的是“本次活动开放报名的场地列表”
- 当前版本不要求用户理解 `ACTIVE / DISABLED` 主数据状态
- `GENERAL / 统一分配` 不展示场地列表，后续分场不在系统内处理

## 12. 人数上限

- `GENERAL` 下配置活动总人数
- `USER_SELECT_COURT` 下配置单场地人数上限
- 必填
- 必须大于 0

## 13. 是否允许候补

- 默认 true

## 14. 活动图文详情

- 选填
- 支持文字段落与图片混排
- 至少允许纯文字、纯图片、图文混排三种内容形态
- 当前版本不支持视频、外链、表格、复杂排版
