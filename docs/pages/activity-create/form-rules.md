# 创建活动页字段规则

## 1. 发布身份

- 必选
- 使用身份卡片单选，不使用下拉框
- 默认选中上次使用身份或默认身份
- 若仅有一个可用身份，仍展示单张身份卡
- 若当前无可用俱乐部身份，展示“去注册俱乐部”

## 2. 活动标题

- 必填
- 长度 1~30

## 3. 活动日期

- 必填
- 不能早于今天

## 4. 活动时间

- 开始时间必填
- 结束时间必填
- 结束时间必须晚于开始时间

## 5. 收费方式

- 必选
- 枚举：FREE / FIXED / AA / OTHER

## 6. 收费金额

- 仅当 chargeMode != FREE 时按创建页规则校验
- 条件必填
- 大于等于 0

## 7. 报名模式

- 必选
- 视觉上归属“报名与场馆区”
- 枚举：`GENERAL / USER_SELECT_COURT`
- `GENERAL` 对外显示为 `统一分配`
- `USER_SELECT_COURT` 对外显示为 `自主选场`
- 切换模式时，不清空公共字段

## 8. 场馆

- 必选
- 来自当前身份下可用场馆列表
- 支持在当前创建流程内新增场馆或编辑场馆定位
- 地图定位默认使用微信小程序原生 `wx.chooseLocation`

## 9. 场地列表

- 仅当 signupMode = USER_SELECT_COURT 时显示
- 至少保留一片场地
- 支持新增、编辑、删除、调整顺序
- 提交的是“本次活动开放报名的场地列表”
- 当前版本不要求用户理解 `ACTIVE / DISABLED` 主数据状态
- `GENERAL / 统一分配` 不展示场地列表，后续分场不在系统内处理

## 10. 人数上限

- `GENERAL` 下配置活动总人数
- `USER_SELECT_COURT` 下配置单场地人数上限
- 必填
- 必须大于 0

## 11. 是否允许候补

- 默认 true

## 12. 活动图文详情

- 选填
- 支持文字段落与图片混排
- 至少允许纯文字、纯图片、图文混排三种内容形态
- 当前版本不支持视频、外链、表格、复杂排版
