# 创建活动页面产品文档

面向对象：UI 设计、前端开发

版本：v2.3

更新时间：2026-04-13

相关文档：

- [创建活动字段规则](./form-rules.md)

## 页面目标

创建活动页用于发布单次活动，并支持“按场地报名”和“统一报名”两种模式。同时承接“再次发布”活动的初始化能力。

当前版本目标：

- 选择发布身份
- 填写活动基础信息
- 配置活动时间与取消截止规则
- 选择场馆
- 配置报名模式
- 配置场地人数或统一报名总人数
- 填写活动描述
- 发布活动
- 再次发布

## 在主链路中的位置

该页面位于主链路中的“正式发布”步骤：

`用户建档 -> 选择发布身份 -> 俱乐部注册 -> 场馆/场地准备 -> 创建活动 -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

本页只承接创建与再次发布，不承接活动发布后的详情、报名和管理动作。

## 前置条件

- 当前用户已完成基础建档
- 当前用户至少拥有一个可用发布身份
- 若用户要以俱乐部身份发布，需先完成俱乐部注册
- 若当前没有可用场馆 / 场地数据，需先补齐主数据

## 页面职责

- 创建新活动
- 再次发布历史活动
- 根据报名模式配置活动级或场地级人数
- 在当前页面内承接场馆与场地主数据补充
- 发布成功后跳转到独立活动详情页

## 非职责边界

- 不承接活动详情展示
- 不承接报名与取消报名
- 不承接活动进行中或已结束后的处理
- 不承接取消活动
- 不承接发布后的调场与调容量

## 入口与回跳

建议入口：

- `activity` 列表中的 `创建活动`
- 首页中的创建活动入口
- 活动详情或我的活动中的 `再次发布`

回跳与承接规则：

- 若在选择发布身份时缺少俱乐部主体，可进入 [俱乐部注册](../club-register/README.md)，成功后返回本页并自动选中新俱乐部身份
- 若缺少场馆 / 场地主数据，可进入 [场馆与场地管理](../venue-court-management/README.md)，保存后返回本页并刷新可选数据
- 发布成功后，统一跳转到 [活动详情](../activity-detail/README.md)

## 核心模块

### 1. 页面标题区

- 普通新建：
  - 标题：`创建活动`
  - 副标题：`按单次活动创建，支持按场地报名或统一报名`
- 再次发布：
  - 标题：`再次发布`
  - 副标题：`已带出上次活动的配置，可直接修改后发布`
- 异常提示条：
  - 原活动引用场地已停用时提示重新确认

### 2. 发布信息区

- 发布身份
- 报名模式
- 活动名称
- 收费方式
- 收费金额
- 收费说明

### 3. 时间信息区

- 活动日期
- 开始时间
- 结束时间
- 停止取消规则

### 4. 报名与场馆区

- 选择场馆
- `管理场馆与场地` 入口
- 按场地报名：
  - 启用场地列表
  - 每片场地人数上限
- 统一报名：
  - 活动总人数

### 5. 活动描述区

- 多行文本输入

### 6. 底部操作区

- `返回`
- `发布活动`

## 字段与状态

### 关键字段

- `ownerType / ownerId`
  - 发布身份
  - 权限口径引用 [角色边界说明](../../global/role-boundaries.md)
- `signupMode`
  - `USER_SELECT_COURT`
  - `GENERAL`
- `title`
  - 必填
- `chargeMode / chargeAmountCents / chargeDesc`
  - 活动级收费配置
- `activityDate / startTime / endTime`
  - 活动统一时间
- `cancelCutoffMinutesBeforeStart`
  - 取消截止规则
- `venueId`
  - 两种模式下都必填
- `courts[]`
  - 仅在 `USER_SELECT_COURT` 模式下提交
- `totalCapacity`
  - 仅在 `GENERAL` 模式下提交
- `descriptionRichtext`
  - 活动描述

### 状态说明

- 活动状态、生命周期、报名状态都不在本页重新定义
- 页面只负责创建前的输入与校验
- 发布后的状态展示统一引用 [活动全链路说明](../../global/activity-lifecycle.md)

## 关键交互

- 普通新建时使用默认表单值初始化
- 再次发布时带回上一场活动的基础配置、时间、收费、报名模式与人数配置
- 切换报名模式时，公共字段不丢失，模式相关配置分别保留
- 切换场馆时刷新当前场馆下的可选场地
- 点击 `管理场馆与场地` 时，在当前流程内补齐主数据后返回本页继续填写
- 点击 `发布活动` 时：
  1. 做字段校验
  2. 生成 `CreateActivityInput`
  3. 创建活动
  4. 成功后跳转活动详情页

## 异常与空状态

- `请填写活动名称`
- `请选择场馆`
- `请至少启用一片场地`
- `XX 的人数上限至少为 1`
- `活动总人数至少为 1`
- `收费金额格式不正确`
- `结束时间需要晚于开始时间`
- `停止取消时间不能晚于活动开始时间`
- 原活动场地已停用时，提示重新确认场地选择

## 验收标准

- 用户可在本页完成活动创建
- 用户可从本页进入俱乐部注册或场馆管理后返回继续填写
- 两种报名模式都可成功发布
- 再次发布可正确带回原活动配置
- 发布成功后统一进入活动详情页
- 本页不再承担报名、取消报名、调场、调容量、取消活动等发布后职责
