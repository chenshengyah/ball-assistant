# 创建活动页面产品文档

面向对象：UI 设计、前端开发

版本：v2.6

更新时间：2026-04-20

相关文档：

- [创建活动字段规则](./form-rules.md)

## 页面目标

创建活动页用于发布单次活动，并支持“统一分配”和“自主选场”两种模式。同时承接“再次发布”活动的初始化能力。

当前版本目标：

- 确认发布身份
- 填写活动基础信息
- 配置活动时间与取消截止规则
- 在报名与场馆区内完成报名模式、场馆与场地相关配置
- 填写活动图文详情
- 发布活动
- 再次发布

## 在主链路中的位置

该页面位于主链路中的“正式发布”步骤：

`完善信息 -> 确认发布身份 -> 俱乐部注册 -> 创建活动（必要时补充场馆与场地） -> 活动详情 -> 报名/取消报名 -> 我的活动回收`

本页只承接创建与再次发布，不承接活动发布后的详情、报名和管理动作。

## 前置条件

- 当前用户已完成基础资料补全
- 当前用户至少拥有一个可用发布身份
- 若用户要以俱乐部身份发布，需先完成俱乐部注册

## 页面职责

- 创建新活动
- 再次发布历史活动
- 在当前页面内确认发布主体
- 根据报名模式配置活动级或场地级人数
- 在当前页面内选择 / 新增 / 编辑场馆并维护本次活动场地列表
- 发布成功后跳转到独立活动详情页

## 非职责边界

- 不承接活动详情展示
- 不承接报名与取消报名
- 不承接活动进行中或已结束后的处理
- 不承接取消活动
- 不承接发布后的调场与调容量

## 入口与回跳

建议入口：

- `activity` 列表中的 `创建活动`
- 首页中的创建活动入口
- 活动详情或我的活动中的 `再次发布`

回跳与承接规则：

- 若在确认发布身份时缺少俱乐部主体，可进入 [俱乐部注册](../club-register/README.md)，成功后返回本页并自动选中新俱乐部身份
- 若当前没有合适场馆，优先在本页直接新增或编辑场馆；仍可进入 [场馆与场地管理](../venue-court-management/README.md) 做补充维护后返回
- 发布成功后，统一跳转到 [活动详情](../activity-detail/README.md)

## 核心模块

### 1. 页面标题区

- 普通新建：
  - 标题：`创建活动`
  - 副标题：`按单次活动创建，支持统一分配或自主选场`
- 再次发布：
  - 标题：`再次发布`
  - 副标题：`已带出上次活动的配置，可直接修改后发布`
- 异常提示条：
  - 原活动引用场地已被删除时提示重新确认

### 2. 发布信息区

- 发布身份卡片组
- 活动名称
- 发起人微信号
- 收费方式
- 收费金额
- 收费说明

发布身份交互要求：

- 使用可见即选的身份卡片，不使用下拉选择器
- 卡片至少展示：
  - 身份名称
  - 身份类型或角色说明
  - 当前选中态
- 若仅有一个可用身份：
  - 仍展示单张身份卡
  - 不隐藏该模块，避免用户不清楚当前发布主体

### 3. 时间信息区

- 活动日期
- 开始时间
- 结束时间
- 停止取消规则

### 4. 报名与场馆区

- 报名模式切换
- 选择场馆
- `新增场馆`
- `编辑场馆定位`
- `管理场馆与场地` 补充入口
- `统一分配 = GENERAL`
  - 仍需选择活动场馆
  - 页面只配置活动总人数
  - 不展示场地列表
  - 后续由组织者线下统一分配，不在系统内落到具体场地
- `自主选场 = USER_SELECT_COURT`
  - 组织者在当前页直接维护本次活动开放报名的场地列表
  - 用户报名时选择对应场地
  - 支持新增、编辑、删除、调整顺序
  - 每片场地人数上限

报名模式交互要求：

- 报名模式放在“报名与场馆区”顶部，作为该区一级控制项
- 切换报名模式时，公共字段不丢失
- 切换到 `统一分配` 时，不清空已选场地，只隐藏场地配置
- 切回 `自主选场` 时，恢复之前场地配置
- 切换报名模式不重置已选场馆
- 当前版本不要求用户先理解场地主数据的 `启用 / 禁用` 状态

### 5. 活动图文详情区

- 图文混排编辑区
- 支持文字段落与图片按顺序组织
- 支持纯文字、纯图片、图文混排三种内容形态
- 当前版本不承接视频、外链、表格和复杂排版

### 6. 底部操作区

- `返回`
- `发布活动`

## 字段与状态

### 关键字段

- `ownerType / ownerId`
  - 发布身份
  - 权限口径引用 [角色边界说明](../../global/role-boundaries.md)
- `signupMode`
  - `GENERAL = 统一分配`
  - `USER_SELECT_COURT = 自主选场`
- `title`
  - 必填
- `chargeMode / chargeAmountCents / chargeDesc`
  - 活动级收费配置
- `activityDate / startTime / endTime`
  - 活动统一时间
- `cancelCutoffMinutesBeforeStart`
  - 取消截止规则
- `venueId`
  - 两种模式下都必填
- `courts[]`
  - 仅在 `USER_SELECT_COURT / 自主选场` 模式下提交
  - 表示本次活动开放报名的场地列表
- `totalCapacity`
  - 仅在 `GENERAL / 统一分配` 模式下提交
- `descriptionRichtext`
  - 活动图文详情富文本内容

### 状态说明

- 活动状态、生命周期、报名状态都不在本页重新定义
- 页面只负责创建前的输入与校验
- 发布后的状态展示统一引用 [活动全链路说明](../../global/activity-lifecycle.md)

## 关键交互

- 普通新建时使用默认表单值初始化
- 再次发布时带回上一场活动的基础配置、时间、收费、报名模式、人数配置与最新图文详情
- 发布身份以卡片方式直接展示，点击卡片即切换发布主体
- 单身份场景下仍展示卡片，但默认选中且不可误解当前主体
- 报名模式在“报名与场馆区”中切换，并即时影响下方配置内容
- 进入页面后优先选择已有场馆；若没有合适场馆，可在当前流程内直接新增场馆或编辑场馆定位
- 新增或编辑场馆定位时，默认调用微信小程序原生 `wx.chooseLocation`
- 切换场馆时刷新当前场馆的可复用场地列表，并允许继续维护本次活动场地列表
- 点击 `管理场馆与场地` 时，作为补充维护入口使用；返回后继续当前创建流程
- `统一分配` 下不展示场地列表，后续分场由组织者线下处理，不在系统内生成场地分配结果
- `自主选场` 下由主办方在当前页直接维护本次活动场地列表，供用户报名时自主选择
- 图文详情在创建阶段与活动基础信息一并提交，不拆成单独二次配置流程
- 点击 `发布活动` 时：
  1. 做字段校验
  2. 生成 `CreateActivityInput`
  3. 创建活动
  4. 成功后跳转活动详情页

## 异常与空状态

- `请填写活动名称`
- `请选择场馆`
- `请至少新增一片场地`
- `XX 的人数上限至少为 1`
- `活动总人数至少为 1`
- `收费金额格式不正确`
- `结束时间需要晚于开始时间`
- `停止取消时间不能晚于活动开始时间`
- 原活动场地已被删除时，提示重新确认场地选择

## 验收标准

- 用户可在本页完成活动创建
- 用户可直接看见所有可用发布身份，并通过卡片完成选择
- 单身份场景下仍可明确看到当前发布主体
- 报名模式与场馆/场地配置在同一区域完成
- 用户可在当前创建流程内直接新增场馆或编辑场馆定位
- `自主选场` 下可在当前页直接新增、编辑、删除活动场地
- `统一分配` 与 `自主选场` 两种模式都可成功发布
- 再次发布可正确带回原活动配置
- 用户可在本页首次录入活动图文详情，并在发布后于活动详情页查看
- 发布成功后统一进入活动详情页
- 文档中不再把“启用 / 禁用场地”作为创建活动主操作
- 本页不再承担报名、取消报名、调场、调容量、取消活动等发布后职责
