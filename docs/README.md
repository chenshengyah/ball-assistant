## 页面 PRD（按主链路）

### P0 主链路

1. [首页](./pages/home/README.md)：公共浏览入口，承接活动推荐、搜索、分类和创建活动入口
2. [活动列表](./pages/activity/README.md)：已并入首页的活动浏览结构说明，保留作为信息架构参考
3. [活动详情](./pages/activity-detail/README.md)：报名、取消报名、调场、调容量、取消活动、再次发布的主承接页

5. [创建活动](./pages/activity-create/README.md)：创建与再次发布，不承接活动发布后的详情和管理动作
6. [俱乐部注册](./pages/club-register/README.md)：补齐俱乐部主体，并承接从创建活动页进入后的返回
7. [俱乐部管理](./pages/club-management/README.md)：从“我的”进入后查看和编辑俱乐部资料，并进入俱乐部场馆管理

9. [我的活动](./pages/my-activities/README.md)：回收“我发布的 / 我报名的”活动集合，并统一进入活动详情
10. [我的](./pages/profile/README.md)：个人入口容器，承接资料、我的活动、创建活动和俱乐部入口

## 页面地图摘要

当前文档口径下，MVP 页面关系固定为：

- 游客浏览：`首页` -> `活动详情`
- 首次触发关键动作但资料未完善：进入 `完善信息 / 资料补全`
- 已建档主办方：`首页` -> `创建活动` -> `活动详情` -> `我的活动（我发布的）`
- 已建档报名者：`首页` -> `活动详情` -> `我的活动（我报名的）`
- 创建活动前缺俱乐部主体：`创建活动` -> `俱乐部注册` -> 返回 `创建活动`
- 创建活动前缺场馆主数据：`创建活动` 内进入 `场馆与场地管理` -> 返回 `创建活动`
- `我的` tab：作为资料与入口容器，承接去 `完善信息 / 资料补全`、`俱乐部注册 / 俱乐部管理`、`我的活动`
- 俱乐部持续维护：`我的` -> `俱乐部管理` -> `编辑俱乐部资料 / 管理场馆与场地`
